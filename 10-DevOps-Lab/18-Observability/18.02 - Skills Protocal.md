# Skill Acquisition Protocol: Typing & Navigation

## 1. Objective
To achieve "frictionless" interaction with the terminal and editor. The goal is **zero cognitive load** on input mechanics, allowing 100% focus on logic and architecture.

## 2. The Toolchain

| Tool | Purpose | Context | LazyVim Key |
| :--- | :--- | :--- | :--- |
| **Ttyper** | **Syntax Fluency** | Practice symbols (`{`, `|`, `$`) and code patterns (Rust/Python). | `<leader>kt` |
| **Gtypist** | **Mechanical Accuracy** | Correcting finger placement and reducing error rates. | `<leader>ks` |
| **VimBeGood** | **Navigation Precision** | Vertical/Horizontal movement without arrow keys. | `<leader>kv` |

## 3. The Routine (The "Algorithm")

### Phase A: Warm-up (Daily - 5 Mins)
*Trigger: Start of day or before a complex coding session.*
1. Launch **Ttyper** (`<leader>kt`).
2. Complete 2 minutes of **Code Syntax** practice (Rust/Linux).
3. **Pass Criteria:** Accuracy > 95%.
    * *If Accuracy < 95%: Proceed to Phase B.*
    * *If Accuracy > 95%: Proceed to Work.*

### Phase B: Correction Drill (As Needed)
*Trigger: Feeling "rusty" or failing Phase A accuracy.*
1. Launch **Gtypist** (`<leader>ks`).
2. Select **Lesson U** (Upper row) or **S** (Symbols) depending on weakness.
3. Drill until 3 consecutive error-free lines.

### Phase C: Navigation Sprint (Weekly)
*Trigger: Friday Review or "Slow" feeling.*
1. Launch **VimBeGood** (`<leader>kv`).
2. Run "Relative Number" drills for 5 minutes.
3. Aim to reduce latency between thought and cursor arrival.

## 4. Telemetry & Logging
Metrics are auto-prompted upon tool exit to ensure consistency.

1. **Ttyper/Gtypist:** Input score into the popup immediately after closing the terminal.
2. **VimBeGood:** Run command `:LogSkill VimBeGood` if a session feels significant.
3. **Review:** Check `18.01 - metrics.md` weekly to visualize trends.
